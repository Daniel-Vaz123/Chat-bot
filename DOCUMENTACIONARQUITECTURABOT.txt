PREGUNTA DIFÍCIL: "¿Por qué usas DeepSeek y no Claude directamente en Bedrock?"
RESPUESTA: DeepSeek tiene mejor relación costo/calidad para respuestas en español.
           Claude en Bedrock se usa para los embeddings (Titan), no para generación.
           La arquitectura es modular: se puede cambiar el LLM sin tocar el RAG.

PREGUNTA DIFÍCIL: "¿Qué pasa si la pregunta no tiene match en la base?"
RESPUESTA: El filtro MaxQueryDistance = 0.75 rechaza el resultado y devuelve un
           mensaje de fallback sin llamar a DeepSeek, ahorrando tokens y costo.

FLUJO DE TRANSCRIPCIÓN (método TranscribeFromFile ~línea 90):
  1. Abre el archivo Ogg/Opus con OpusOggReadStream
  2. Decodifica paquete a paquete con OpusDecoder(16000, 1)
  3. Convierte short[] → byte[] (little-endian PCM 16-bit)
  4. Alimenta VoskRecognizer.AcceptWaveform(pcmBytes, length)
  5. Extrae texto del JSON final: recognizer.FinalResult() → {"text": "..."}

CÓDIGO CLAVE (VoskTranscriptionService.cs ~línea 95-110):
  var decoder = new OpusDecoder(16000, 1);
  var oggIn   = new Concentus.Oggfile.OpusOggReadStream(decoder, fileIn);
  using var recognizer = new VoskRecognizer(_model, 16000.0f);
  recognizer.SetMaxAlternatives(0);
  recognizer.SetWords(false);
  while (oggIn.HasNextPacket) {
      var pcmShorts = oggIn.DecodeNextPacket();
      var pcmBytes  = new byte[pcmShorts.Length * 2];
      Buffer.BlockCopy(pcmShorts, 0, pcmBytes, 0, pcmBytes.Length);
      recognizer.AcceptWaveform(pcmBytes, pcmBytes.Length);
  }

PREGUNTA DIFÍCIL: "¿Por qué Vosk y no Whisper de OpenAI?"
RESPUESTA: Vosk corre 100% offline. Whisper requiere enviar el audio a la API de OpenAI.
           Para un gimnasio, las notas de voz pueden contener datos personales sensibles.
           Vosk también tiene latencia menor al no depender de red externa.

FLUJO COMPLETO DEL WEBHOOK (WhatsAppController.cs):
  1. Validar que payload.From no esté vacío
  2. Resolver texto: audio → VoskTranscriptionService | texto → payload.Body
  3. Detectar saludo/menú → enviar mensaje de bienvenida (con caché 7 días)
  4. Llamar ChatbotService.AskQuestionAsync(texto) → RAG + DeepSeek
  5. Enviar respuesta con IChannelAdapter.SendMessageAsync(userId, respuesta)

RESPUESTA: El puente Node.js usa whatsapp-web.js para conectarse a WhatsApp Web,

¿Qué hace el bot?
  Responde preguntas de socios de gimnasio por WhatsApp usando RAG + IA generativa.

¿Qué es RAG?
  Retrieval-Augmented Generation: busca el contexto relevante en una base vectorial
  ANTES de pedirle a la IA que genere la respuesta. Evita alucinaciones.

¿Cómo funciona el vector store?
  AWS S3 Vectors almacena vectores float32 de 1024 dimensiones.
  La búsqueda usa distancia coseno. TopK=1 para texto, TopK=5 para imágenes.

¿Qué modelo de IA se usa?
  Embeddings: Amazon Titan Embed Text v2 (texto) + Titan Embed Image v1 (imagen)
  Generación: DeepSeek Chat (deepseek-chat), temperatura 0.3

¿Cómo llegan los mensajes de WhatsApp?
  Twilio → ngrok (puerto 5000) → POST /api/whatsapp → WhatsAppController.cs

¿Cómo se procesan las notas de voz?
  Vosk (offline, español) decodifica Ogg/Opus → PCM 16kHz → texto. Sin nube.

¿Qué pasa si la pregunta no tiene respuesta en la base?
  MaxQueryDistance=0.75 filtra el resultado. Si distancia > 0.75, respuesta de fallback.

¿Cuántos escenarios conversacionales hay?
  3 implementados (PropositoAnoNuevo, AtletaEstancado, Desertor) de 16 definidos.
  Cada uno tiene 3 handlers: TOFU, MOFU, BOFU.

¿Hay mensajes automáticos?
  Sí. GymTriggerBackgroundService corre cada hora y detecta inactividad > 15 días,
  post-primera-clase 24h, hitos de gamificación y formularios abandonados.
